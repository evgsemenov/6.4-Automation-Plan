#### 1.7 Запись на курс неавторизованного пользователя
- Открыть страницу профессии [Тестировщик](https://netology.ru/programs/qa), не авторизуясь на сайте
- Проскроллить страницу вниз до формы записи на курс
- В поле "Имя" ввести имя из тестовых данных
- В поле "Телефон" ввести телефон из тестовых данных
- В поле "электронная почта" ввести email из тестовых данных
- Нажать кнопку "Записаться"
  
  ОР:
  
  Появляется сообщение об успешной записи на курс

#### 1.8 Запись на курс авторизованного пользователя

  ОР:

#### 1.11 Отправка пустой формы записи

- Проскроллить страницу вниз до формы записи на курс

#### 1.12 Валидация данных поля "Имя" при заполнении формы записи на курс профессии 
- Открыть страницу профессии [Тестировщик](https://netology.ru/programs/qa)

- В поле "Имя" ввести кириллические символы не короче двух символов, с использованием символов "ё", "-", "Space"
  
  ОР:
  
  Значение принимается, никаких комментариев под полем не появляется

  ОР:
  
  Поля "Имя" подсвечивается с комментарием "Обязательное поле".

  ОР:
  
  Поля "Имя" подсвечивается с комментарием "Должно состоять из букв"

- В поле "Имя" ввести любые цифры
  
  ОР:

### 2. Инструментарий

- JUnit5 v5.7.2- тестовый фреймворк
- Selenide v5.23.2 - реализация взаимодействия с UI
- Cucumber JVM: JUnit v6.11.0 - интеграция ручных тестов с автоматическими и их поддержка
- Allure JUnit 5 v2.14.0 - отчетность и аналитика
- Github Actions - непрерывная интеграция с автоматической проверкой публикуемых билдов

### 3. Перечень необходимых разрешений/данных/доступов

Для реализации тестирования необходимо предоставление учетных данных тестового пользователя с заполненными полями "Имя" и "Телефон",
либо предоставление прав для создания тестового пользователя.

Также необходимо согласование с отделом продаж формата отправляемых тестовых данных
во избежание обработки отделом продаж заявок, отправленных в ходе тестирования. 

### 4. Перечень и описание возможных рисков при автоматизации

Возможна обработка отделом продаж "ложных" заявок, отправленных в ходе тестирования.

### 5. Перечень необходимых специалистов для автоматизации

Инженер по ручному тестированию - 1 человек

Инженер по автоматическому тестированию - 1 человек

### 6. Интервальная оценка с учётом рисков (в часах)

18 часов
